# 9장. 실용주의 프로젝트 p.377 ~ p.406 2022-04-05
### 책에서 기억하고 싶은 내용
- 모든 테스트가 끝날 때까지는 코딩이 끝난 게 아니다.
- 버그를 심어 놓고 테스트를 테스트하라.
- 코드 커버리지만 올리지 말고 상태 조합을 테스트하라.
- 버그는 한 번만 잡아라.
- 자신의 작품에 서명하라.

### 읽고 떠오르는 생각
- 정말 의미가 있는 팁들이 많았다. 사용자가 필요할 떄 제공하라라는 말처럼 업적을 남겨야 한다는점에 감탄하면서 본 것 같다.

### 소감 3줄 요약
- 작고 안정적인 팀을 유지하라.
- 실천하려면 계획하라.
- 사용자에게 필요할 때 제공하라
